# 修改接口实现以符合文档要求

## 1. 问题分析
通过对比接口文档和当前实现，发现以下差异：

### 1.1 CharactersController - POST /api/characters
- **请求参数**：文档中使用 `current_location`（下划线命名），但实现中只支持 `currentLocation`（驼峰命名）

### 1.2 ConversationsController - POST /api/conversations
- **请求参数**：文档中使用 `current_location`（下划线命名），但实现中只支持 `currentLocation`（驼峰命名）
- **返回结果**：文档中使用 `current_location`（下划线命名），但实现中返回 `location`

### 1.3 UsersController - POST /api/users
- **返回结果**：错误响应格式与文档不一致，文档中没有定义错误响应格式

## 2. 修改计划

### 2.1 修改 CharactersController
- **POST /api/characters**：同时支持 `current_location` 和 `currentLocation` 两种命名方式

### 2.2 修改 ConversationsController
- **POST /api/conversations**：同时支持 `current_location` 和 `currentLocation` 两种命名方式
- **返回结果**：将 `location` 字段改为 `current_location`

### 2.3 修改 UsersController
- **POST /api/users**：统一响应格式，确保与文档一致

## 3. 修改说明
- 所有修改都将保持向后兼容性，同时支持驼峰命名和下划线命名
- 返回结果将严格按照文档格式输出
- 确保所有接口的请求参数、返回结果和状态码都与文档一致

## 4. 测试建议
- 修改后需要测试所有接口，确保功能正常
- 测试不同命名方式的请求参数，确保都能正确处理
- 验证返回结果格式与文档一致
